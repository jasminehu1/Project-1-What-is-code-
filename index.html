5.1 What is the Relationship Between Code and Data?

E-mails, tweets, Facebook posts, Word documents, Fitbit exercise tracker, these are all things that data comes from. Data management is the problem that programming is supposes to solve but because technology is evergrowing, there are now computers everywhere. Which requires more programming. Then there's more data. Then more programming. It's a vicious endless cycle.

5.2 Where Does Data Live?

The most prevalent is the relational database, using a language called SQL, for Structured Query Language. It's basically a library, each item has a ID and can be found in this database with that ID. Then there's Oracle, Oracle makes you pay thousands of dollars to use its commercial enterprise database, but more and more of the world runs on free software databases.

5.3 The Language of White Collars

Most programmers think of Java when it comes to corporate programming. Java is a programming language that was born at Sun Microsystems, the product of a team led by a well-regarded programmer named James Gosling. Some people complain but Java gets the job done.

Java is a big language but it came with a ton of built-in classes to deal with a database, documents, mathematics and network services. Java also automatically generated documentation, now you could run a tool called javadoc, and it would make you Web pages that listed all the classes and methods.There were a lot of Java manuals, workshops and training seminars, and certifications. Programmers can take classes and tests to be officially certified in many technologies. Java programmers had an especially wide range to choose from.

5.4 Briefly on the Huge Subject of Microsoft

Microsoft does not like Java, it was worming their way into their money and enterprises. Don’t ever count Microsoft out. Statically typed, scripting, data-driven, functional—name your ambiguous adjective, and Microsoft will sell something that delivers that to you, and you can write Windows code in it and live a Windows life. Microsoft products can be as good as or better than anything else on the market.

5.5 Liquid Infrastructure

Google and Amazon are high-preforming companies that have cultures that know how to make software. They have whole departments dedicated to testing.

Academic researchers often produce things that basically work but don’t have interfaces. They need to prove their theses, publish, and move on to the next thing. People in the free software community often code to scratch an itch and release that code into the digital commons so that other people can modify and manipulate it. While more often than not this process goes nowhere, over time some projects capture the imagination of others and become part of the infrastructure of the world.

Java, interestingly, profits from all this. It’s designed for big corporate projects and has the infrastructure to support them. It’s also a useful language for midsize tasks. Eventually, people realized that if they didn’t like the Java language, they could write other languages that compile to Java “bytecode” and run on the Java virtual machine.

Languages can be updated to use code from other languages; this is the case for Clojure, which is a Lisp language adapted by Rich Hickey. This transmutable property of code makes its infrastructure very fluid.

5.6 Off the Shelf

You’re making a system that will integrate with the systems around it and your company is a set of such systems, nothing is truly off the shelf. There are tools and packages and libraries, and if you have any wit at all you already use well-documented, free code for things such as e-mail validation, but that obviates only so much.

5.7 What about Javascript?

Javascript makes webpages interactive. Javascript became important when people came to understand Ajax, which turns web pages into software and enables the Internet to be not just a publishing platform but an app-delivery platform.
